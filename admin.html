Security
Recruiter access code and admin password
Credentials are set as Railway environment variables — RECRUITER_CODE, ADMIN_PASS, and GATE_SUBTITLE.
Go to Railway → your service → Variables to set or update them. Trigger a redeploy after changing.
GATE_SUBTITLE controls the subtitle text shown on the security gate (e.g. "Software Engineer"). Default is "GATE_SUBTITLE_PLACEHOLDER" if unset — set it in Railway Variables.
When testing new credentials, open an Incognito window or clear pf_role from Session Storage in DevTools.
Recruiter Access Code
Print this code at the bottom of your resume PDF so only real recruiters have it.
Admin Password
Set ADMIN_PASS in Railway → Variables. Redeploy after changing. Do not store the password here.
Publish
Save config directly to the server — changes are live instantly, no GitHub push needed
How it works: Clicking "Save & Publish" writes data/config.json directly to the Railway Volume. Changes are live on the site immediately — no redeploy, no git push needed.

Instant local preview: "Save Changes" (top-right) also updates your browser's localStorage preview immediately.
Publish to Server
